   2. Recursive Character TextSplitter (MOST USED) -> PDFs, Websites, RAG apps, Production chatbot, Docs AI  
      Meaning preserve karta hai.  
      Difference b/w Character and Recursive Character TextSplitter

      ## Character vs. Recursive Character TextSplitter

      | Feature                | Character TextSplitter      | Recursive Character TextSplitter    |
      |------------------------|----------------------------|-------------------------------------|
      | Cutting method         | Blind cutting              | Smart, meaning-preserving cutting   |
      | Word splitting         | Words can break            | Tries to keep words and sentences   |
      | Use case               | Basic, small projects      | Production apps, better for docs    |
      | Usage frequency        | Rarely used                | Most commonly used                  |
      | Retrieval quality      | Poor (bad retrieval)       | Good (better retrieval)             |

      Example:
      Original Text:
      "React is a frontend library. It is created by Meta. Hooks are useful."

      Character TextSplitter (chunk_size=20):
      - Chunk 1: "React is a fronte"
      - Chunk 2: "nd library. Hook"
      - Chunk 3: "s are useful."

      Recursive Character TextSplitter (chunk_size=20):
      - Chunk 1: "React is a frontend"
      - Chunk 2: "library."
      - Chunk 3: "Hooks are useful."
   
   3. TokenTextSplitter: OpenAI / Claude apps, Large prompts, Cost optimization, Advanced RAG
       Why use it? -> Character-based text splitters are unpredictable in terms of token count and model cost, whereas LLMs process text as tokens. Using a TokenTextSplitter gives you precise control over costs and chunk sizes, making it ideal for large prompts and cost optimization.

       React is a frontend library created by Meta
       Code: TokenTextSplitter (chunk_size = 8)
       ["React"," is"," a"," frontend"," library"," created"," by"," Meta"] = 8 tokens.
